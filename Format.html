Meshフォーマットについて

IDの始まりについて

 節点や要素のIDは、C++の都合上、プログラム内では全て「0始まり」「連番」で記述しています。メッシュを読み込んだ際、非連番の場合は読み取った順にゼロ始まりで連番になるようにIDを書き換えます。もとの節点・要素IDが重要な場合はご注意ください。メッシュを読み込んだ際に、元のIDを記憶する機能があります。ConvMeshクラスのメンバとして、「node_map」「ele_map」に元IDと振りなおしIDの対応を記憶しているので、必要な場合は活用してください。現在、デフォルト機能として対応関係を書き出す機能は作っていません。

 メッシュ書き出す場合、ID=0に対応していないフォーマットは+1して書き出します。書き出し時に+1するかどうかは下記のとおりです。

・+1して書き出すフォーマット： unv, atls, neu

・0始まりのまま書き出すフォーマット： vtk, jpmmesh, jpmmesh_e

・書き出し非対応フォーマット： msh, mesh

 メッシュ操作を行う場合、節点や要素のIDをjsonで支持する場合、「1始まり」をデフォルトとしています。 多くのメッシュビューアは「1始まり」だと思うので、表示されている値をそのまま利用すればよいはずです。

 ただ、paraviewは0始まりに対応しており、vtkは0始まりで処理しているので、現状paraviewでvtkを読み取った際は、デフォルト記述だと変になります。 paraview利用時のみ、表示されているIDを+1して利用するか、「NUMBER_ID」コマンドでゼロ始まりモードに変更してください。

自作メッシュフォーマットについて

 本OSS独自フォーマットとして、「jpmmesh」「jpmmesh_e」形式を作っています。「jpmmesh_e」は辺も書き出すフォーマットです。FEMなどで使うように、予めフォーマットの段階から隣接情報や（電磁気に使う辺の情報）を書き出すFEM特化フォーマットです。（隣接情報などを作るため、他の書き出しより前処理に少し時間を要します）

jpmmeshフォーマット

Node #ラベル
node_num #節点数
id x y z  #節点IDと、その座標をx, y, zの順で。以降、node_numだけ繰り返し
....

Element #ラベル
ele_num #全要素数
3D  #3D要素の数
2D  #2D要素の数
1D  #1D要素の数
Info: 0 303 1   #ラベル、要素ID、要素タイプ、材料ID
17 0 8 18 2698 2681 2689 2699  #節点IDを要素の節点数分列挙
.... #以降、ele_numだけ繰り返し

NeighbourElements #ラベル
ele_num #全要素数
ELE: 0 303 4 #ラベル、要素ID、要素タイプ、隣接要素の数 (以降、隣接要素の数だけ繰り返す)
303 1 #隣接要素のタイプ、隣接要素のID
,,,,
.....#以降、ele_numだけ繰り返し

BoundNodeList #ラベル
bn_num #メッシュ端にある節点の数
id #メッシュ端節点のID
.....#以降、bn_numだけ繰り返し

MinimumEdge #ラベル
0.001 #最小の辺の長さ
